# StructuralPattern
A clean, well‑structured .NET solution demonstrating all GoF Structural Design Patterns with real‑world examples, UML diagrams, and runnable code samples. Includes Adapter, Bridge, Composite, Decorator, Facade, Flyweight, and Proxy patterns implemented using modern C# practices

Structural Design Patterns in .NET – Complete Examples
This repository contains a clean, professional, and fully runnable collection of GoF Structural Design Patterns, implemented using modern C# and .NET best practices.
Each pattern is isolated in its own folder with:
- Clear folder structure
- Interfaces + concrete implementations
- Real‑world examples
- A simple console demo
- Pattern summary and usage notes
This project is ideal for developers who want to learn, teach, or reference structural patterns in real applications.

🧱 Included Structural Patterns
✔ Adapter Pattern
Convert one interface into another so incompatible systems can work together.
Example: Integrating a third‑party SMS service into your own notification interface.
✔ Bridge Pattern
Decouple abstraction from implementation so both can vary independently.
Example: Different message types (Email/SMS) with different senders.
✔ Composite Pattern
Treat individual objects and groups of objects uniformly.
Example: File/folder hierarchy.
✔ Decorator Pattern
Add responsibilities to objects dynamically without modifying their code.
Example: Logging, caching, or validation wrappers.
✔ Facade Pattern
Provide a simplified interface to a complex subsystem.
Example: A single “OrderService” that hides multiple internal services.
✔ Flyweight Pattern
Share common object state to reduce memory usage.
Example: Reusing character glyphs in a text editor.
✔ Proxy Pattern
Control access to another object (lazy loading, caching, security).
Example: Virtual proxy for loading large images.
